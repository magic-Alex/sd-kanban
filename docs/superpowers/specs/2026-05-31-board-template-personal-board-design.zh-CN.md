# 统一看板模板与个人任务看板设计规格

## 背景

SD Kanban 当前已经支持项目总看板、我的任务、任务拖拽、任务分配、项目成员关系、用户管理和通知。现有项目总看板可以在项目内部拖动任务卡片改变状态；我的任务页面目前只是以看板外观展示个人任务，还没有真正支持列间拖拽。项目成员添加的后端接口已经存在，但前端缺少项目成员管理入口。

为了让个人任务看板能跨项目混合展示任务，同时避免不同项目看板列不一致带来的拖拽歧义，后续所有项目统一使用同一套全局看板列模板。项目不再单独维护不同的列布局，模板由系统设置统一管理。

## 目标

- 新增全局看板列模板，所有项目使用同一套状态列。
- 新增系统设置入口，管理员可以维护统一看板模板。
- 新建项目自动复制当前全局模板列。
- 模板变更同步到所有项目看板。
- 默认列改为中英文双提示：待办（Backlog）、就绪（Ready）、进行中（In Progress）、测试（Testing）、完成（Done）。
- 新增项目编号和项目颜色，创建项目时必填，项目编号全局唯一。
- 我的任务改成一组统一状态列的个人看板，支持拖拽任务到不同状态。
- 个人看板拖拽只改变任务状态，不改变任务所属项目。
- 任务卡片展示项目编号、项目名称和项目颜色标识。
- 项目详情页补齐项目成员管理，项目负责人可以添加和移除项目成员。

## 非目标

- 不开放用户自助注册，用户仍由管理员创建。
- 不支持每个项目单独自定义看板列。
- 不在本次实现复杂权限角色矩阵；项目成员仍维持 owner/member 两级。
- 不引入实时协同或 WebSocket。
- 不实现批量迁移任务状态的复杂向导；删除有任务的模板列直接禁止。

## 核心产品规则

### 全局模板

系统只有一套看板列模板。每个模板列包含：

- 模板键 `templateKey`：稳定标识，例如 `BACKLOG`、`READY`、`IN_PROGRESS`、`TESTING`、`DONE`。
- 中文名 `nameZh`。
- 英文名 `nameEn`。
- 展示名：由中文名和英文名组合，例如 `完成（Done）`。
- 颜色 `color`。
- 排序 `sortOrder`。
- WIP 限制 `wipLimit`。
- 是否完成列 `isDone`。

项目实际仍保留 `board_columns`，任务继续绑定具体项目列 `column_id`。项目列新增 `template_key`，用于和全局模板列对应。这样既能保留现有任务表结构，也能让个人看板按模板状态统一分组。

### 模板同步

- 新建项目时，按当前模板创建该项目的所有项目列。
- 修改模板列名称、英文名、颜色、WIP、完成状态时，同步更新所有项目中相同 `template_key` 的列。
- 调整模板排序时，同步更新所有项目列排序。
- 新增模板列时，为所有项目追加对应项目列。
- 删除模板列前检查所有项目中对应列是否存在任务；只要有任务，禁止删除并提示先迁移任务。
- 删除无任务模板列时，同时删除所有项目中对应列。

### 项目字段

项目新增：

- 项目编号 `projectCode`：必填、唯一、建议大写短码，例如 `SD-KB`。
- 项目颜色 `projectColor`：必填，用于个人看板卡片色标。

已有项目迁移时自动回填：

- `projectCode` 使用 `PRJ-<项目ID>`。
- `projectColor` 使用默认色板按项目 ID 稳定分配。

### 个人任务看板

我的任务页面改为一组统一状态列：

- 数据只包含当前登录用户作为负责人的未归档、未删除任务。
- 列来自全局模板，而不是某个项目。
- 每张卡片显示项目编号、项目名称和项目颜色。
- 拖拽卡片到目标模板列时，前端提交 `taskId + targetTemplateKey + sortOrder`。
- 后端读取任务所属项目，再找到该项目中同 `templateKey` 的实际项目列，最后更新任务 `columnId` 和排序。
- 更新后任务仍属于原项目，项目 ID、项目编号、项目颜色不改变。

如果某个项目缺少目标模板列，说明模板同步异常。后端拒绝移动，并返回“该项目缺少目标状态列，请先同步看板模板”。

### 项目成员与任务分配

用户创建仍在“用户管理”中完成。

项目成员管理放在项目详情页：

- 项目成员列表展示成员、角色、加入时间。
- 项目负责人可以添加已有启用用户为成员。
- 项目负责人可以移除普通成员。
- 项目负责人不能移除自己。
- 项目负责人可以转交项目负责人，沿用已有后端能力。
- 任务创建和任务详情中的负责人下拉框只展示该项目成员。
- 如果用户不在项目成员列表中，不能被分配任务。

## 后端设计

### 数据迁移

新增迁移：

- 给 `projects` 增加 `project_code`、`project_color`。
- 给 `board_columns` 增加 `template_key`。
- 新增 `board_column_templates` 表。
- 初始化默认模板列。
- 回填已有项目编号、颜色和已有列的 `template_key`。

默认模板映射：

- sort 0 或名称 `Backlog` -> `BACKLOG` -> 待办（Backlog）
- sort 1 或名称 `Ready` -> `READY` -> 就绪（Ready）
- sort 2 或名称 `In Progress` -> `IN_PROGRESS` -> 进行中（In Progress）
- sort 3 或名称 `Testing` -> `TESTING` -> 测试（Testing）
- sort 4 或名称 `Done` -> `DONE` -> 完成（Done）

### 服务分层

新增系统设置相关模块，保持 controller、service、service impl、repository 分层：

- `settings/controller/BoardTemplateController`
- `settings/service/BoardTemplateService`
- `settings/service/impl/BoardTemplateServiceImpl`
- `settings/repository/BoardColumnTemplateRepository`

项目服务创建项目时不再硬编码默认列，而是读取模板服务生成项目列。

看板服务返回卡片时补充项目上下文：

- `projectCode`
- `projectName`
- `projectColor`
- `columnTemplateKey`

任务服务新增个人看板移动能力：

- `PATCH /api/tasks/{taskId}/personal-position`
- 请求体：`targetTemplateKey`、`sortOrder`
- 内部复用现有任务位置更新逻辑，但目标列由后端按任务所属项目解析。

### 权限

- 模板设置仅管理员可访问。
- 项目成员管理仅项目负责人可变更。
- 项目成员可查看项目成员列表。
- 个人任务拖拽要求当前用户是项目成员，并且任务当前负责人是当前用户；项目负责人如需调整任务状态，仍从项目总看板操作。

## 前端设计

### 导航与页面

新增“系统设置”导航，仅管理员可见。

系统设置包含“看板模板”页：

- 模板列列表。
- 新增列、编辑列、调整顺序、删除列。
- 每列显示中文名、英文名、颜色、WIP、完成列标识。
- 删除有任务的列时显示后端返回的阻止原因。

项目创建表单新增：

- 项目编号。
- 项目颜色。

项目详情页新增：

- 项目信息卡：编号、颜色、负责人、成员数。
- 项目成员列表。
- 添加成员弹窗。
- 移除成员按钮。

我的任务页改造：

- 去掉按项目/按状态切换。
- 展示统一模板列。
- 复用项目总看板的卡片和拖拽体验。
- 卡片增加项目徽标：颜色点或色条 + 项目编号。
- 空列显示轻量空状态。

### 组件复用

- `BoardColumn` 抽出更通用的拖拽列组件能力，支持项目总看板和个人看板复用。
- `TaskCard` 增加可选项目上下文显示。
- `ProjectDetailView` 承担成员管理入口。
- 新增 `SystemSettingsView` 或 `BoardTemplateSettingsView`。

## 错误处理

- 项目编号重复：提示“项目编号已存在”。
- 项目编号为空或格式不合法：提示“请输入项目编号，建议使用大写字母、数字和短横线”。
- 项目颜色为空或格式不合法：提示“请选择项目颜色”。
- 添加已存在成员：提示“该用户已经是项目成员”。
- 添加不存在或禁用用户：提示“请选择有效用户”。
- 删除有任务的模板列：提示“该状态下仍有任务，请先迁移任务”。
- 个人看板拖拽到异常列：提示“目标状态不可用，请刷新后重试”。
- 模板同步失败：事务回滚，不产生部分项目更新。

## 测试策略

### 后端

- 模板初始化与迁移测试。
- 新建项目复制模板列测试。
- 模板名称、颜色、排序、WIP 修改同步测试。
- 删除有任务模板列被阻止测试。
- 删除无任务模板列同步删除测试。
- 项目编号唯一性测试。
- 项目成员添加、移除、权限测试。
- 个人任务按模板列返回测试。
- 个人任务拖拽按任务所属项目解析目标列测试。
- 非负责人不能在个人看板移动任务测试。

### 前端

- 项目创建表单校验项目编号和颜色。
- 项目详情成员管理渲染与提交。
- 系统设置模板列表、编辑、删除阻止提示。
- 我的任务只有一组统一看板列。
- 我的任务拖拽调用个人位置 API。
- 任务卡片显示项目编号和项目颜色。
- 管理员才显示系统设置入口。
