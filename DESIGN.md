# SD Kanban 设计系统

本文件用于指导 SD Kanban 的前端界面实现。设计目标不是营销页，而是一个工程团队每天会打开、扫视、拖动、筛选、创建任务的敏捷工作台。

## 1. 视觉主题与气质

SD Kanban 采用“工程团队精确型工作台”风格：

- 参考 Linear 的克制和精确：低噪声、清晰层级、快速定位任务。
- 参考 Airtable 的结构化数据表达：筛选、表格、状态、负责人、列数据一眼可读。
- 参考 Notion 的柔和工作区表面：浅色背景、轻边框、不过度装饰。

界面要安静、专业、耐看。避免大面积渐变、营销式英雄区、装饰性卡片堆叠、过度圆角、单一蓝紫色主题。

## 2. 颜色角色

使用语义色变量，不在组件里散落硬编码颜色。

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| `--color-canvas` | `#f6f8fb` | 页面背景 |
| `--color-surface` | `#ffffff` | 主面板、弹窗、任务卡 |
| `--color-surface-muted` | `#f9fbfd` | 看板列、浅色输入背景 |
| `--color-border` | `#dce4ec` | 标准边框 |
| `--color-border-soft` | `#e8edf3` | 分隔线 |
| `--color-text` | `#17212b` | 主文本 |
| `--color-muted` | `#607083` | 辅助文本 |
| `--color-subtle` | `#8997a8` | 占位、说明、弱信息 |
| `--color-primary` | `#0f766e` | 主操作、当前状态 |
| `--color-primary-soft` | `#d9f5f1` | 主色浅底 |
| `--color-info` | `#2563eb` | 信息、需求、链接 |
| `--color-success` | `#16a34a` | 完成、启用 |
| `--color-warning` | `#d97706` | WIP、到期、注意 |
| `--color-danger` | `#dc2626` | 删除、停用、逾期 |
| `--color-violet` | `#7c3aed` | 依赖、泳道、统计辅助 |

功能色不能只靠颜色表达，必须配合文字，例如“逾期”“超限”“停用”。

## 3. 字体与排版

- 中文优先使用系统 UI 字体：`Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", sans-serif`。
- 字号阶梯：12、13、14、16、18、22、28、32。
- 正文最小 14px，移动端输入和正文尽量 16px，避免移动浏览器自动缩放。
- 标题使用 700 或 800，标签/按钮使用 700，正文使用 400 或 500。
- 行高正文 1.5 到 1.65，界面标签 1.3 到 1.45。
- 字距保持默认，不使用负字距。

## 4. 布局原则

- 桌面端：固定左侧导航 + 主内容工作区。导航宽度 244px 到 260px。
- 主内容：顶部页头承载页面标题、说明和主要操作。列表、看板、图表在下方。
- 看板：列是横向滚动工作区，每列宽度稳定，任务卡高度由内容自然撑开，但核心信息顺序固定。
- 移动端：顶部品牌/账号区 + 横向导航或紧凑导航，主内容单列，按钮不拥挤，不出现水平页面滚动。
- 弹窗/抽屉：背景使用遮罩，抽屉宽度桌面约 560px，移动端占满宽度。关闭、保存、删除等操作保持可见。
- 页面 section 不做大卡片嵌套，只有重复项、表格容器、弹窗、任务卡使用卡片。

## 5. 组件规则

### 按钮

- 主按钮：青绿色实心，用于每屏唯一主操作，如“新增任务”“保存任务”“创建用户”。
- 次按钮：白底或浅底边框，用于筛选、取消、编辑、恢复。
- 危险按钮：浅红底红字，用于删除、停用。
- 所有按钮最小高度 40px，移动端触控目标不小于 44px。
- 禁用态要降低透明度并保留可读文本。

### 表单

- 必须有可见 label，placeholder 只做补充。
- 错误信息放在相关表单区域附近，并使用 `aria-live="polite"`。
- 搜索和筛选区域要紧凑，桌面横排，移动端单列。

### 任务卡

任务卡信息顺序：

1. 标题
2. 类型与优先级
3. 负责人
4. 故事点、截止日期、清单进度

逾期日期使用红色浅底标签。清单进度使用“清单 1/3”文字。

### 看板列

- 列头显示色点、列名、任务数和“+ 任务”按钮。
- 空列显示明确空状态。
- 支持 WIP 时，超限用红色提示，正常用青绿色提示。

### 通知与抽屉

- 通知面板右侧滑出，未读项用浅色背景和边框强调。
- 任务抽屉将操作按钮放在标题区，详情、检查清单、描述、验收标准、标签、评论、动态分区展示。

## 6. 动效与状态

- 交互动效 120ms 到 180ms，只使用 `transform` 和 `opacity`。
- hover 提升边框或背景，不做夸张阴影。
- `prefers-reduced-motion` 下取消非必要动效。
- loading 状态用文字或按钮禁用，不让用户重复提交。

## 7. 可访问性

- 所有图标按钮必须有 `aria-label`。
- 表单控件必须有 label 或明确的 `aria-label`。
- focus ring 不能移除，统一使用主色外环。
- 正文与背景对比至少达到 WCAG AA。
- 键盘 Tab 顺序要符合视觉顺序。

## 8. 响应式

| 断点 | 行为 |
| --- | --- |
| `> 1100px` | 桌面工作台，左侧固定导航，内容密度较高 |
| `700px - 1100px` | 导航变为顶部流式布局，面板减少列数 |
| `< 700px` | 单列内容，按钮换行，任务抽屉满宽，表格行变卡片 |

移动端禁止页面级横向滚动；看板列本身可以横向滚动。

## 9. 禁止事项

- 不使用 Figma 输出作为运行时依赖。
- 不使用 emoji 当图标。
- 不使用大面积紫蓝渐变、装饰圆球或营销式 hero。
- 不在卡片里再套大卡片。
- 不用 placeholder 代替 label。
- 不让按钮文字在小屏幕溢出。
